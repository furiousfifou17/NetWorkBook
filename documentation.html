  • Imprimer

    Nouveau



    Entrez ici les informations sur l'établissement.

  • Nom du lycée
  • Trigramme : Le trigramme de l'établissement, il sera utilisé dans le nom des piles réseaux.
  • Route par défaut : Entrez ici l'IP de destination pour tous les réseaux du coeur de réseau.
  • Ip DHCP : Elle servira pour les commandes de DHCP relay.
  • Ip TFTP : Elle sera utile pour la commande job save-conf afin de faire des sauvegardes automatiques des configurations.
  • Ip NTP : Utile pour la mise à l'heure des piles.
  • PortUDP WOL : Port UDP utilisé pour le wake on lan.
  • Mot de passe admin : Mot de passe pour le compte admin de switchs.
    • Les paramètres FreeRadius non obligatoire :
  • FreeRadius adresse IP : L'ip de votre serveur Freeradius en cas de mobilité de vlan.
  • Freeradius phrase secréte : Phrase secrète partagée entre Freeradius et les switchs.
    • Snmpv3 (monitoring des switch en snmpv3):
  • Utilisateur
  • Mot de passe
  • Phrase privée
    • Il sera possible après de modifier ces informations sauf le mot de passe admin des switchs et la partie SNMP. Une fois les informations renseignées, Valider.

    Il ne reste plus qu'à choisir un nom de fichier pour votre base de donnée.

    Les onglets


    Informations


    Dans cet onglet, vous pouvez modifier les informations du lycée


    VLans


    Dans cette onglet, vous retrouvez toutes les informations sur vos vlans, ainsi que leurs gestions, ajout, modification et suppression.

    Pour ajouter un vlan cela se passe dans le formulaire en bas.
    Enregistrez les informations suivantes :

  • Id du vlan : Le numéro du vlan.
  • Nom du vlan : Le nom du vlan.
  • Couleur du vlan : Cliquer sur pour choisir une couleur, cette dernière est juste utilisée sur le cahier réseau, elle n'a aucun effet sur les scripts.
  • Passerelle : La passerelle du vlan même si ce dernier n'est pas routé par le coeur de réseau.
  • Masque : choisissez dans la liste déroulante le masque au format CIDR. L'adressage du sous réseau, première adresse, dernière adresse et le Broadcast ce calcul automatiquement.
  • Router par le niveau 3 : En choisissant Oui, le sous réseau sera routé par le coeur de réseau.
  • DHCP Relay : En choisissant Oui, le vlan bénéficiera des options pour le DHCP Relay au niveau du coeur.
  • UDP Relay : En choisissant Oui, le vlan laissera passer les requêtes de WOL.
  • Qos auto VOIP : , à placer sur votre vlan de VOIP.
  • Jumbo trames :, en utilisant cette options vous activez les jumbo 9600 sur le vlan comme pour le clan storage par exemple.
  • Multicast (PPMS) :, choisir Non pour le vlan PPMS.

    • Une fois les informations enregistrées, cliquez sur Ajouter.

    Pour Modifier ou supprimer un vlan, faites un clic droit dans le tableau sur le vlan en question. Un menu contextuel vous permet de modifier ou supprimer ce vlan.

    En choisissant Modifier le formulaire ajout se transforme en Modifier un vlan , effectuez vos modifications et validez.

    En chosissant Supprimer le vlan est supprimé, ATTENTION!!! aucune confirmation n'est demandé, l'action est immédiate.

    Que se soit pour l'ajout, la modification ou la suppression d'un vlan, il vous est demandé si vous voulez envoyer l'action en SSH sur toutes les piles.ATTENTION ne pas faire cette action pendant la création du cahier réseau car vos piles ne sont pas encore configurée.

    Piles


    Sur cet onglet, nous allons gérer la plus grosse partie du cahier réseau les switchs.
    En haut à gauche trois boutons, pour la création, la modification et la suppression de piles de switchs.

    Nouvelle Pile

    En cliquant sur ce bouton une nouvelle fenêtre s'ouvre pour la création d'une pile.

  • Nom : Entrez le nom à donner à la pile ou un switch seul, n'indiquez pas le trigramme, il est pré rempli.
  • Emplacement : Ici l'emplacement de la pile
  • Ip : L'ip de management de la pile
  • Vlan : Le vlan de management de la pile
  • Stack : Gardez Non si il s'agit d'un switch seul ou basculez sur Oui dans le cas d'une pile de switchs. Dans ce cas-là deux boutons + et - vont apparaitre sur la droite et le champ domaine vsf va se compléter automatiquement avec le dernier octet de l'adresse IP.
  • Case à cocher Coeur de réseau : cochez cette case si il s'agit de la pile du coeur de réseau afin que les interfaces de routage lui soit attribués.

    • Le tableau sert à constituer une pile, à gauche le numéro du switch dans la pile, à droite une liste déroulante avec les modèles de switchs. Avec le plus cela ajoute un switch et ainsi de suite.

    Une fois la pile constituée cliquez sur "Valider". La fenêtre se fermera et votre pile apparait dans la liste déroulante "Pile".

    Modifier Pile

    Cette option vous permet de modifier une pile existante. L'interface est sensiblement le même que celui d'une nouvelle pile

    Supprimer Pile

    Sert à détruire une pile

    Liste déroulante Pile

    Elle permet de choisir la pile que l'on fait apparaitre dans le tableau à droite. Dans la liste déroulante "Switch" en dessous le switch de la pile à afficher. Un bouton "Actualiser" est présent au cas où cela ne s'actualise pas tout seul.
    Dans la fenêtre "Informations de la pile" on retrouve les informations de la pile et un bouton "Générer script VSF" ce dernier sert à créer les petits scripts de configuration du stack vsf.

    Rechercher

    Cette option permet de rechercher une prise RJ et de focusser le tableau dessus.

    Modifier

    En cliquant sur l'un des ports du switch les champs de la fenêtre "Modifier" vont se remplir.
    Voici les options possibles :

  • VSF : Cochez cette case si il s'agit d'un port choisi pour le stack VSF.
  • Vlan par défaut : Choisisez ici le vlan par défaut du port.
  • Authentification MAC :Cette case reste cochée si vous n'avez pas défini de serveur Radius et de phrase secrète. En cochant cette case le port serra configurer pour utiliser l'authentification par mac.
  • Vlans tagués : Cochez ici les vlans qui seront tagués sur le port.
  • BPDU Protection : Cochez cette case pour activer la "BPDU Protection", utile si vous savez qu'il y'a un "switch de table" derrière ce port. Attention n'activez pas cette option sur un port de liaison inter-baie ou un port d'une borne WiFi.
  • Destination : Entrez ici le nom de la prise RJ dans la salle. Cette saisie est semi-automatique si vous avez importez vos prises en csv, sinon la prise s'enregistrera dans la liste des prises au fur et à mesure.
  • Commentaire : Entrez ici le commentaire ou pas que vous souhaitez, cela peut être le nom de la salle par exemple

    • A noter que ces deux dernières options donneront un nom à vos interfaces dans la conf sous la forme "Destination | Commentaire". Ce qui peut être pratique quand vous faite un "show interface x/x".

    En cliquant sur le bouton "Modifier", vous avez la possibilité d'envoyer la modification directement sur le switch en SSH.

    ATTENTION ne pas utiliser cette option pendant la création du cahier réseau vos piles ne sont pas encore configurées. Vous serrez invité à entrer votre login et mot de passe pour pouvoir envoyer la commande.

    L'onglet Agrégations

    Ici un bouton "Nouvelle agrégation" vous permet de créer une agrégation. Cela ouvre une nouvelle fenêtre

    Ici le nom de l'agrégation est automatique en fonction du modèle de switch et du nombre d'agrégations déjà existantes sur la pile choisie.

  • Pile : Liste déroulante pour choisir la pile sur laquelle l'on créé une agrégation.
  • Interface 1 et Interface 2 : List déroulante pour chosir les interfaces.

    • Il ne reste plus qu'à valider pour créer l'agrégation, vous la retrouverez dans l'onglet Piles sur les piles concernées.


    Le tableau en dessous du bouton "Nouvelle agrégation" affiche l'ensemble des agrégations de votre réseau.

    L'onglet Prises

    Ici l'inventaire de vos prises RJ. Une case à cocher permet de voire uniquement les prises non brassées, ce qui peut être utile pour savoir les prises encore libres dans une salle par exemple.

    Un bouton "Ajouter une prise" ouvre une fenêtre pour ajouter une prise RJ dans votre inventaire

    Le champ Destination affiche une liste en fonction de la saisie ce qui est utile pour éviter d'enregistrer deux fois la même prise. Le champ commentaire affiche aussi une liste pratique pour choisir un lieu existant.

    La fenêtre modifier permet de modifier ou supprimer une prise RJ. Pour cela cliquez sur la prise voulue dans le tableau.

    L'onglet Routes statiques

    Cet onglet est là pour créer des routes statiques autres que les interfaces de routages ou les routes par défauts. Par exemple les routes pour les réseaux d'un PFSense.

    Ips fixes

    Cet onglet est là pour stocker les adresses IP fixes de votre réseau. Cela ne rentre pas en compte dans la création des scripts.

    En haut à gauche une liste déroulante pour choisir le sous réseau où vous voulez enregistrer une IP. En choisissant un numéro de vlan le tableau à droite se met à jour avec les informations déjà enregistrées. Un formulaire permet d’ enregistrer, modifier ou supprimer une IP.

    Plan de Masse

    Ici nous pouvons faire glisser chaque baie sur un plan de masse. Pour changer la position d'une baie il suffit de la re selectionner dans la liste déroulante et de la faire glisser au bonne endroit.

    Si aucun plan de masse n'a été importer dans le logiciel vous verrez le message suivant :

    Cliquez sur Ok et vous serrez rediriger vers le formulaire d'information pour uploader un plan au format PNG, JPG ou BMP.
